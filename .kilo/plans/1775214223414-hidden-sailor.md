# CI/CD Workflow Plan for Giftly

## Current State Analysis
- Existing CI workflow at `.github/workflows/ci.yml`
- Runs on push to main/develop branches and pull requests
- Uses Python 3.9, 3.10, 3.11 matrix
- Installs dependencies and runs tests with coverage
- Already uses pytest.ini configuration implicitly
- Tests located in `tests/` directory as requested

## User Request Summary
Create a CI/CD workflow that:
1. Runs tests on GitHub once a commit is pushed
2. Uses the folder root tests containing .py unit test files
3. Uses pytest.ini as configuration for pytest
4. Keep it simple ("nothing more")

## Plan Evaluation
The existing workflow largely satisfies the requirements but may need adjustment based on interpretation of "once a commit is pushed":
- Current: Runs on push AND pull_request
- Possible interpretation: Run only on push (not pull_request)

## Proposed Changes
Since a workflow already exists and mostly meets requirements, the plan is to:

1. **Option A (Minimal Change)**: Keep existing workflow as-is since it:
   - Runs on push (fulfills core requirement)
   - Uses tests/ directory
   - Uses pytest.ini configuration
   - Runs unit tests (.py files in tests/)

2. **Option B (Exact Interpretation)**: Modify workflow to run only on push (not pull_request) if "once a commit is pushed" excludes pull requests

3. **Option C (Simplify)**: Create a simpler workflow matching the "nothing more" request

## Recommended Approach
Given the existing workflow is functional and comprehensive, I recommend **Option A** with verification that:
- pytest.ini is properly configured and used
- Tests in tests/ directory are discovered and run
- Workflow triggers on push events

## Implementation Steps (if changes needed)
1. Verify current workflow uses pytest.ini correctly
2. Confirm test discovery works for tests/ directory
3. Ensure workflow triggers on push events
4. Optionally simplify by removing pull_request trigger if desired
5. Optionally reduce complexity if "nothing more" means minimal workflow

## Files to Review/Modify
- `.github/workflows/ci.yml` - Main workflow file
- `pytest.ini` - Pytest configuration (verify it's correct)
- `tests/` directory - Verify test structure

## Validation Criteria
- Workflow runs on git push
- Discovers and executes .py test files in tests/ directory
- Uses pytest.ini for test configuration
- Reports test results clearly
